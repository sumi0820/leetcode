#### アプローチ 1

気づいたこと/ memo

- RFC によると`"@"word"`のような local-part も許容されるため、単純に`@`で split するのは本来はよくなさそう
  - `Each emails[i] contains exactly one '@' character.`とあるので、今回は一旦考慮しない方向で検討する
  - 実際やるなら`matchAll()`もしくは`lastIndexOf`で最後の`@`の位置を取得し、`slice()`で分断は可能そう
    - 今回は`lastIndexOf()`を使うのが直感的で読みやすいと判断し採用
- step1 の各変数名が分かりづらかったので他の方の PR を参考にさせてもらった
- 今まで string 繋ぎでは`+`を使っていたが、改めて GPT に確認すると文字列を返す場合は template literals の方がベターとのことだった
  - 変数に number 型がきた場合に`+`を使用すると意図せず加算される可能性があるため
  - また、将来的に結合すべき変数がや文字列が増えた場合にその分`+`を増やさないといけず冗長になるため

```typescript
function numUniqueEmails(emails: string[]): number {
  const formattedEmails = emails.map((e) => formatEmail(e));
  return new Set<string>(formattedEmails).size;
}

const formatEmail = (email: string): string => {
  const lastAtPosition = email.lastIndexOf("@");
  const local = email.slice(0, lastAtPosition);
  const domain = email.slice(lastAtPosition + 1);

  const localWithoutPlus = local.slice(0, local.indexOf("+"));
  const normalizedLocal = localWithoutPlus.replace(/[.]/g, "");

  return `${normalizedLocal}@${domain}`;
};
```

気づいたこと

- for 文を２重でループさせて検証する方法を推してきた
  - 先のアプローチでは`map`と`set`両方を使用していることで全体で 2×N 回の走査 と N 回の関数呼び出しをしているため 1 回あたりの処理コストが高いことがわかった
    - ループ 1：map が emails を 1 件ずつ処理 → 新しい配列を生成 → 各要素ごとに コールバック呼び出し（関数呼び出し）
    - ループ 2：new Set(formattedEmails) が配列をもう一度走査して要素を登録
  - 一方 for 文の方は、「ループ数（定数 2→1）＋関数コール回数（2×→1×）＋割り当て（配列生成なし）」になる
  - 加えて、JS の関数呼び出しは数百万回繰り返すと確実に無視できない程の overhead が生じる、 `map()`は新しい配列を作成するためにメモリを確保しにいくためその観点からも for で元の配列を呼び出す方が低コストのよう
- また、GPT と壁打ちをしている中でアプローチ１のバグを指摘される
  - `local.indexOf("+")`がない時に`-1`になってしまうため、それ以降のコードが意図しない箇所で slice されてしまう可能性がある

解法

```typescript
export function numUniqueEmails(emails: string[]): number {
  const seen = new Set<string>();

  for (const email of emails) {
    const at = email.lastIndexOf("@");
    const domain = email.slice(at + 1);

    // local を1パスで正規化（'+'で打ち切り、'.'は無視）
    let local = "";
    for (let i = 0; i < at; i++) {
      const ch = email[i];
      if (ch === "+") break;
      if (ch !== ".") local += ch;
    }

    seen.add(`${local}@${domain}`);
  }
  return seen.size;
```

## 解答

- GPT の案がパフォーマンス良さそう＋自分の発想になかったためこれを書けるようにする

```typescript
export function numUniqueEmails(emails: string[]): number {
  const seen = new Set<string>();

  for (const email of emails) {
    // ドメイン部分を取得
    const atPosition = email.lastIndexOf("@");
    const domain = email.slice(atPosition + 1);

    // ローカル部分をチェック
    let local = "";
    for (let i = 0; i < atPosition; i++) {
      const char = email[i];
      if (char === "+") break; // +の場合はその時点でループを終了
      if (char !== ".") local += char; //.以外をlocal変数に追加
    }
    seen.add(`${local}@${domain}`);
  }
  return seen.size;
}
```
